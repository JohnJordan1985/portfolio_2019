WhatsApp Live Location

SUMMARY: Designing and user testing a "Live Location" feature for WhatsApp... six months later they released this feature with a very similiar implementation, validating my approach...

I paired up with another product designer for the project, and I proposed we prototype a live location feature for WhatsApp – it was something that I felt the application was missing, based on my usage of it.

I took ownership of the wireframe prototyping and user-testing, while my colleague was to create the high-fidelity prototypes, once we had established the structure and layout of the live location feature.

When proposing new product features, I’ve always found it easier to start with problems that need solving, and work backwards to a feature that would solve them.

I’d used the static location-sharing feature of WhatsApp, but found it inconvenient that you had to stay where you are were – or remember to update your location if you moved. Being able to share your live location would avoid these two issues.

My colleague agreed it was a useful feature, so we decided to begin work: I would be taking care of the initial prototyping and all the user testing, with my colleague preparing the high-fidelity visual prototypes.

After deciding on an initial set of user tasks, I sketched out a few interface layouts, and prepared initial task flows for each user task, which I then used to structure the initial wireframe prototype.
